# 05_Cases-vs-Guarantee

## 目的
Case と Guarantee の関係を整理し、**ケースがどのような保証要求を持つか、また保証理論の検証材料としてどう機能するか**を定義してください。

---

## この文書で扱う範囲
扱う内容：

- Case と Guarantee の接続関係
- ケースごとの保証要求
- 保証密度・保証衝突・保証分解可能性
- ケースを通じた Guarantee 理論の検証

扱わないこと：

- Guarantee Space 全体理論の再定義
- 数学的形式化の深掘り
- 実際のテスト仕様詳細

---

## 前提
`50_guarantee` では、保証単位や保証空間が定義されています。  
`70_cases` では、それをケースへ適用し、**どのような構造がどのような保証要求を生むのか**を観測できるようにする必要があります。

---

## 必須観点
必ず以下を含めてください。

1. **Case と Guarantee の関係**
   - なぜ Case 単位で Guarantee を考えるのか
   - Scope 単位やノード単位では捉えきれない点

2. **保証要求の発生要因**
   - 制御構造の複雑性
   - データ依存
   - I/O依存
   - 外部依存
   - 境界曖昧性
   - 影響伝播性

3. **保証密度**
   - 保証項目が多いケース
   - 保証確認コストが高いケース
   - 保証量と構造の関係

4. **保証衝突**
   - 複数の保証要件が相互に緊張関係を持つケース
   - どのような構造で衝突しやすいか

5. **保証分解可能性**
   - 保証を局所化しやすいケース
   - 分解しにくいケース
   - 分割移行との関係

6. **Guarantee Space との接続**
   - ケースを保証集合としてどう表現できるか
   - ケース比較にどう使えるか
