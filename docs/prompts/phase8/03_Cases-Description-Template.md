# 03_Cases-Description-Template

## 目的
`70_cases` における各ケースを、**同一のテンプレートで記述できるようにするための記述様式**を設計してください。  
この文書の目的は、ケースの比較可能性・再利用性・検証可能性を確保することです。

---

## この文書で扱う範囲
扱う内容：

- ケース記述テンプレートの必要性
- テンプレート項目の設計
- 各項目の意味
- 構文層／構造層／判断層をまたぐ記述方式
- 比較可能性を高めるための記述原則

扱わないこと：

- 実際のケース内容の作成
- 個別テンプレートの記入例の大量列挙
- スコアリングや数値評価モデルの詳細

---

## 前提
Case は「事例」ではなく「判断検証単位」です。  
したがってテンプレートも、単なる説明用フォーマットではなく、**移行判断・保証割当・影響分析に利用可能な構造的記述様式**として設計してください。

---

## 必須観点
必ず以下を含めてください。

1. **テンプレートが必要な理由**
   - 記述のばらつきを抑えるため
   - ケース比較を可能にするため
   - 保証と判断を接続しやすくするため

2. **テンプレート項目**
   少なくとも以下を含めること：

   - ケースID
   - ケース名
   - 対象範囲
   - 対象構文
   - 構造要約
   - 主な AST / CFG / DFG 関与要素
   - 制御構造上の特徴
   - データ依存上の特徴
   - I/O依存
   - 外部依存
   - 境界の明瞭性
   - 想定される Guarantee
   - 判断上の論点
   - 想定リスク
   - 分割可能性
   - 変更影響の方向
   - 検証コスト / 観測コスト
   - 備考

3. **層別記述**
   - 構文層
   - 構造層
   - 判断層
   を分けて記述する設計にすること

4. **比較可能性**
   - どの項目が比較軸になるか
   - 定性的項目と定量化候補項目を分けること

5. **テンプレート運用原則**
   - 省略可能項目の扱い
   - 不明情報の扱い
   - 判定不能条件の書き方

---

## 出力形式
以下の構成で Markdown として出力してください。

- タイトル
- 1. 文書の目的
- 2. ケース記述テンプレートが必要な理由
- 3. テンプレート設計原則
- 4. テンプレート項目一覧
- 5. 各項目の意味
- 6. 構文層／構造層／判断層の分離
- 7. 比較可能性を高めるためのルール
- 8. 不明情報・判定不能の扱い
- 9. 推奨テンプレート（そのまま使える形）
- 10. まとめ

---

## 要求品質
- 実運用でコピペして使えるテンプレートを含めること
- 説明だけでなく、研究用様式として再利用可能にすること
- 後続の比較・評価・検証に接続できる構造にすること
