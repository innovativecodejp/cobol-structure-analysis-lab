# 10_Cases-Mapping-to-AST-CFG-DFG-Guarantee-Decision

## 目的
Case を、既存の研究モデルである **AST / CFG / DFG / Guarantee / Decision** に対応付け、  
`70_cases` を研究全体の接続層として位置づける文書を作成してください。

---

## この文書で扱う範囲
扱う内容：

- Case と AST の関係
- Case と CFG の関係
- Case と DFG の関係
- Case と Guarantee の関係
- Case と Decision の関係
- モデル横断マッピングの意義

扱わないこと：

- 各モデルの全面再定義
- 実装コード変換方法

## 前提
`70_cases` は独立章ではなく、これまでの研究成果を適用・接続・検証する層です。  
したがって、本書では Case を中心にして、既存モデル群をどう横断的に結び付けるかを整理してください。

---

## 必須観点
必ず以下を含めてください。

1. **Case と AST**
   - Case がどの AST ノード群を含むか
   - Case を AST の単位と同一視しないこと
   - AST は観測基盤であり、Case は判断単位であること

2. **Case と CFG**
   - Case がどの制御部分構造に対応するか
   - 分岐・反復・段落遷移の観点

3. **Case と DFG**
   - Case 内部のデータ依存
   - Case 境界をまたぐ依存
   - 共有データによる結合

4. **Case と Guarantee**
   - Case が要求する保証集合
   - 保証密度・保証衝突・保証分解可能性

5. **Case と Decision**
   - Case をどう判断軸へ接続するか
   - 判断理由の説明単位としての Case

6. **モデル横断マッピングの意義**
   - Case を中心にした統合理解
   - 研究全体の整合性
   - 将来のケースベース推論や診断支援への布石
