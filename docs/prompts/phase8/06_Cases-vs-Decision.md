# 06_Cases-vs-Decision

## 目的
Case と Decision の関係を整理し、**ケースをどの判断軸で評価し、どのように移行可否・要分割・要代替などの判断へ接続するか**を定義してください。

---

## この文書で扱う範囲
扱う内容：

- Case と移行判断の関係
- ケース評価に必要な判断軸
- 判断可能条件と判定不能条件
- ケースを通じた判断説明性

扱わないこと：

- Decision 理論そのものの全面再定義
- 実案件の商流・コスト見積
- 実装手段の選定

---

## 前提
`60_decision` では移行判断軸が設計されています。  
`70_cases` では、ケース単位でその判断を適用し、**なぜその判断になるかを説明可能にすること**が目的です。

---

## 必須観点
必ず以下を含めてください。

1. **Case と Decision の関係**
   - なぜ Case 単位で判断するのか
   - ノード単位や構文単位だけでは足りない理由

2. **主要判断軸**
   少なくとも以下を含めること：

   - 可移行
   - 要分割
   - 要代替
   - 要保留 / 判定不能

3. **判断に必要な観測情報**
   - 制御構造
   - データ依存
   - I/O依存
   - 外部依存
   - 境界の明瞭性
   - 保証要求
   - 影響範囲

4. **判定不能条件**
   - 構造情報不足
   - 外部仕様不足
   - 保証要件未定
   - 境界不明

5. **判断説明性**
   - なぜそのケースが可移行なのか
   - なぜ要分割なのか
   - なぜ要代替なのか
   - なぜ判定不能なのか

6. **Decision モデルの検証**
   - ケース群を使って判断一貫性を確認する考え方
   - 判断軸の不足をあぶり出す考え方
