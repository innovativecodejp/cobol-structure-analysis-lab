# 2026-03-30_03_BasicBlockAndControlRegion

## 🎯 今日の目的
- `docs/30_cfg/03_Basic-Block-and-Control-Region.md` に沿い、CFG を局所単位として扱う **基本ブロック** と **制御領域** を要約する。
- statement / basic block / paragraph の **意図的なズレ** と、Scope 候補への接続可能性を押さえる。

## 🛠 実施内容
- 基本ブロックを **単一入口の最大直列実行片** とする定義を整理した。
- **単一出口は基本ブロックに課さない** 方針と、より大きな領域での SE/SX が **閉包議論** に接続することを確認した。
- 制御領域を、入口・出口・内部遷移で説明できる **部分グラフのまとまり** として整理した。
- statement（AST）・basic block（CFG）・paragraph/section（業務参照）の三層対照表を要約した。
- paragraph ≠ region、section は境界候補だが制御閉包と一致しない場合がある、という **ズレ** を明文化した内容を拾った。

## 🔍 結果
- 制御解析の典型単位（BB）と、まとまりの単位（region）が分離され、**Scope 候補の制御的根拠** を語れるようになった。
- region は構文の塊ではなく **制御のまとまり** である、という立場が固定された。

## 💡 学び
- 移行・保証では「どの単位で閉じるか」が問題になり、BB/region はその **最初の構造指標** になる。
- Scope は単一 region 写像に還元できない場合があり、**業務閉包と制御閉包の非一致** が中核論点だと理解した。

## 🧠 思考整理
- 支配・厳密閉包は `08` に回し、本稿は **入口・出口・内部遷移の直観** で足場を作る段階だと位置づけた。
- 一 paragraph が複数 BB に跨り region が多出口、というとき **保証分割と経路爆発** が同時に悪化しうる、というリスク観を保持した。

## 🧭 抽象レベル
- CFG 上の **分割（basic block）** と **領域まとまり（control region）** の理論定義。

## ⚖ 移行判断への示唆
- BB は影響の局所化・カバレッジ単位の基礎。region は **まとめて移行できるか／分割が要るか** の初見指標。
- Guarantee は region 内経路集合に命題を貼りうる、Decision は SE/SX を欠く領域・多出口を難易度に写しうる。
