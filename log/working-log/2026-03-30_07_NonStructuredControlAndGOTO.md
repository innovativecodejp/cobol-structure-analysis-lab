# 2026-03-30_07_NonStructuredControlAndGOTO

## 🎯 今日の目的
- `docs/30_cfg/07_NonStructured-Control-and-GOTO.md` に沿い、非構造制御を **道徳対象ではなく CFG 上の構造破壊の程度** として理論化する。
- 非構造辺の **意味分類** と **構造回復可能性** を、判断接続層の説明負債に接続する。

## 🛠 実施内容
- 非構造制御を、単一入口・単一出口の合成規則だけでは **CFG の経路構造として再構成しにくい** 寄与と定義した。
- 非構造辺を **領域横断・入口／出口多重化・ループ破壊** 等の性質で定義し、類型表に整理した。
- 構造回復（局所／大域）を **移行・検証・説明コストの上界** を示す記号として位置づけた。
- COBOL 論点（paragraph 横断、section 境界、fall-through、EXIT と jump の混線）を列挙した。
- IR・`05`/`06`・`08`・判断接続層との接続を整理した。

## 🔍 結果
- `GO TO` 等を **同一 CFG 上のエッジ分類** で扱い、構造化部分との差異を明示する立場が言語化された。
- 「回復不能＝悪」ではなく **コスト見積りの構造指標** として書けるようになった。

## 💡 学び
- merge 手前への非構造辺集中は **path 説明の破綻** とセットで観測するのが実務的。
- ループ破壊は `06` の反復スキーマ **単純モデルの失効** として記録すると後続と整合する。

## 🧠 思考整理
- 非構造度は、**説明不能経路の増加 → 置換言語 idiom への写像の非一意性** という鎖で移行リスクに効く。

## 🧭 抽象レベル
- 非構造制御は **CFG を SESE 的合成で再構成しにくくする辺クラス** と、その **回復可能性** として整理した。

## ⚖ 移行判断への示唆
- スコープ分割は制御閉包と業務閉包の不一致が拡大しやすい。
- 入口・出口の組合せ増はテスト設計と Guarantee の整合を難しくする。
